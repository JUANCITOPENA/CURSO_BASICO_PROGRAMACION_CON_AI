Introducción: Un reto de innovación digital… y de aprendizaje acelerado
=============================================================================================================================================

Un grupo de estudiantes recibe un desafío: construir en menos de 5 horas una tienda online completa para la venta de vehículos, con diseño responsive, carrito de compras, simulación de pago y generación de factura en PDF. El reto no es menor: deben cumplir estándares modernos de accesibilidad, SEO, SEM y experiencia de usuario, utilizando únicamente HTML5, CSS3, JavaScript puro, Bootstrap 5 y Font Awesome.

En paralelo, un desarrollador senior con años de experiencia decide enfrentarse al mismo reto, pero sin apoyo externo, confiando únicamente en su conocimiento acumulado y en su práctica profesional.

Ambos parten del mismo punto: un archivo JSON con los datos de los vehículos. Sin embargo, no cuentan con las mismas herramientas. Los estudiantes se apoyan en la inteligencia artificial para guiar cada paso, entender las tecnologías, documentar el proceso y desplegar el proyecto en GitHub Pages y Vercel. El profesional, en cambio, recurre a su dominio técnico y a su intuición para resolver cada detalle.

El resultado revela una nueva realidad: la IA no sustituye al desarrollador, pero sí transforma la velocidad y accesibilidad del desarrollo web. Permite que más personas —con o sin experiencia previa— puedan crear, experimentar y crecer en campos como el ecommerce y la programación frontend. Mientras el experto asegura calidad y precisión, los estudiantes logran ejecutar el proyecto y comprender las piezas del código gracias a la guía inteligente.

PROMPT PARA CREAR UNA TIENDA SIMULADA: GARAGE ONLINE

Crea tres archivos separados: index.html, style.css, y script.js para un sitio web de ecommerce altamente interactivo y sofisticado de venta de vehículos.  El sitio debe cumplir al 100% con los siguientes requisitos, utilizando solo HTML5, CSS3, y JavaScript puro (ES6+), además de Bootstrap 5 y Font Awesome para la iconografía.  Los datos de los vehículos deben ser consumidos dinámicamente del siguiente archivo JSON:  https://raw.githubusercontent.com/JUANCITOPENA/Pagina_Vehiculos_Ventas/refs/heads/main/vehiculos.json

Requisitos Generales:

1.  **Semántica y Accesibilidad:**

    *   Implementar un HTML5 completamente semántico (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, etc.).
    *   Cumplir con las pautas WCAG (mínimo nivel AA) utilizando atributos ARIA apropiados para navegación y elementos interactivos.  Asegurar que la navegación sea fácilmente comprensible por lectores de pantalla.

2.  **Diseño Adaptativo:**

    *   Utilizar Bootstrap 5 para un diseño completamente adaptativo que funcione correctamente en múltiples navegadores (Chrome, Firefox, Safari, Edge) y dispositivos (escritorio, tabletas, móviles).

3.  **Frontend Dinámico:**

    *   **HTML (index.html):**
        *   Estructura base: `<!DOCTYPE html>`, `lang="es"`, metadatos, enlaces a Bootstrap y Font Awesome (desde CDN).
        *   Estructura del `header`:
            *   Logo interactivo "GarageOnline" con un icono de coche (`<i class="fas fa-car">`).
            *   Menú fijo con enlaces: "Inicio" y "Vehículos".  Usa clases de Bootstrap para la navegación (`navbar`, `navbar-expand-lg`, `nav-link`, etc.).
            *   Barra de búsqueda (`<input type="text" id="searchInput">`).
            *   Icono de carrito con un contador dinámico (`<i class="fas fa-shopping-cart">`, `<span id="cartCount">`).  El contador debe ser una insignia (badge) de Bootstrap.
            *   Agrega una imagen de fondo, tipo backgroudn 100% expandida en el Header. (https://t4.ftcdn.net/jpg/02/82/00/75/360_F_282007508_wdCUP7hUMNK1Cuzj7XmOcFmzyzJ0Nnp9.jpg) ancho 100% y altura 50%.
            *   Estructura de `main`:
            *   Sección de Inicio (`<section id="inicio">`) con un mensaje de bienvenida.
            *   Sección de Vehículos (`<section id="vehiculos">`):
                *   Título "Nuestros Vehículos".
                *   Contenedor dinámico (`<div class="row" id="productsContainer">`) donde se cargarán las tarjetas de los vehículos.
                *   Mostrar un spinner de carga inicial (usar el spinner de Bootstrap) mientras se cargan los datos.  El spinner debe tener el ID "loadingSpinner".
        *   Estructura del `footer`:
            *   Información de derechos de autor.
        *   Modales (usar componentes modales de Bootstrap):
            *   Modal de Cantidad (`#quantityModal`):  Se abre antes de añadir un producto al carrito.  Debe tener un campo de entrada numérica (`<input type="number">` con `id="quantityInput"`) para la cantidad, un botón "Cancelar" y un botón "Añadir al Carrito" (`id="addToCartBtn"`).
            *   Modal del Carrito (`#cartModal`):  Muestra los ítems del carrito (marca, modelo, cantidad, subtotal), el total del carrito (`<span id="cartTotal">`), un botón "Seguir viendo vehículos" y un botón "Pagar" (`id="checkoutBtn"`) que abre el modal de pago.  El contenido del carrito se debe insertar dinámicamente en un elemento con `id="cartItems"`.
            *   Modal de Pago (`#paymentModal`):  Simula un proceso de pago.  Debe tener campos básicos (nombre, número de tarjeta,  no se requiere validación compleja), un botón "Cancelar" y un botón "Procesar Pago" (`id="processPaymentBtn"`).

**Ventana Modal al hacer clic en la imagen o boton ver detalle:**

Cuando se cargue la imagen con todas las tarjetas (cards), los detalles del vehículo y el botón "Añadir al carrito", al hacer clic en dicho botón debe abrirse una ventana modal con un ancho y alto suficientemente grandes para visualizar en detalle la foto del vehículo (usando su URL). Debajo de la imagen deben mostrarse los detalles organizados en una lista, junto con el botón "Añadir al carrito".


Se añade un solo "oyente" de clic al contenedor principal que nunca cambia: #productsContainer.
Este "oyente" está siempre activo.
Cuando haces clic dentro de ese contenedor, el oyente se pregunta: "¿El elemento exacto en el que se hizo clic fue un botón con la clase .viewDetailsBtn?".
Si la respuesta es sí, entonces ejecuta el código para abrir el modal.

    *   **CSS (style.css):**
        *   Estilos base para la página (fuente, padding para la barra de navegación fija).
        *   Estilos para la barra de navegación (fondo oscuro, texto blanco, logo con negrita).
        *   Estilos para las tarjetas de producto:
            *   Margen inferior, sombra suave.
            *   Transición suave al hacer hover (elevar ligeramente la tarjeta).
            *   Altura fija para las imágenes (`object-fit: cover`).
            *   Usar Flexbox para alinear verticalmente el contenido de la tarjeta (título, descripción, precio) y empujar el botón "Añadir al Carrito" al final de la tarjeta.
            *   Altura fija para el título de la tarjeta (para que quepan hasta dos líneas) y para la descripción (una línea), usando `overflow: hidden`.
        *   Estilos para los botones (colores primarios de Bootstrap, hover).
        *   Animación "pulse" para el contador del carrito (cambio de escala sutil).
        *   Estilo para el spinner de carga.

    *   **JavaScript (script.js):**

        *   **Carga de Datos:**
            *   Usar la Fetch API (`async/await`) para leer los datos de los vehículos desde el archivo JSON proporcionado.
            *   Manejar posibles errores durante la carga (mostrar un mensaje de error en `productsContainer`).
            *   Almacenar los datos cargados en una variable `vehiclesData`.
            *   Ocultar el spinner una vez cargados los datos.

        *   **Mostrar Vehículos (`displayVehicles` function):**
            *   Limpiar el contenido de `productsContainer` antes de insertar las tarjetas.
            *   Iterar sobre el array de vehículos (`vehiclesData`).
            *   Para cada vehículo, crear una tarjeta (elemento `div` con clases de Bootstrap para el grid: `col-md-4`, `col-sm-6`, `mb-4`, y `card`, `h-100`).
            *   Dentro de la tarjeta:
                *   Imagen del vehículo (`vehicle.imagen`, usar `loading="lazy"`).
                *   Título (`vehicle.marca` y `vehicle.modelo`).
                *   Categoría (`vehicle.categoria`).
                *   Tipo (Quitar los emojis del campo `tipo`).
                *   Precio (`vehicle.precio_venta`, formateado con `toLocaleString()`).
                *   Botón "Añadir al Carrito" (clase `btn-primary`,  `data-codigo` con el valor de `vehicle.codigo`).
            *   Añadir la tarjeta a `productsContainer`.
            *   Después de crear *todas* las tarjetas, llamar a la función `addAddToCartListeners`.

        *   **Filtrado (`filterVehicles` function):**
            *   Obtener el texto de búsqueda (`searchInput.value`, convertir a minúsculas).
            *   Filtrar `vehiclesData` usando `filter()`.  El filtro debe buscar coincidencias (en minúsculas) en los campos `marca`, `modelo` y `categoria`.
            *   Llamar a `displayVehicles` con los vehículos filtrados.

        *   **Gestión del Carrito:**
            *   Variable `cart` (array) para almacenar los ítems del carrito.  Cada ítem debe ser un objeto con las propiedades: `imagen`,`logo`,`codigo`, `marca`, `modelo`, `precio`, `quantity`.
            *   `addAddToCartListeners` function:
                *   Seleccionar todos los botones con la clase `addToCartBtn`.
                *   Añadir un event listener de tipo `click` a cada botón.
                *   Dentro del listener:
                    *   Obtener el `codigo` del vehículo desde el atributo `data-codigo` del botón (convertir a número con `parseInt`).
                    *   Encontrar el vehículo correspondiente en `vehiclesData`.
                    *   Llamar a `showQuantityModal` con el vehículo seleccionado.
            *   `showQuantityModal` function:
                *   Crear una instancia del modal de Bootstrap (`#quantityModal`).
                *   Establecer el valor inicial de `quantityInput` a 1.
                *   Asignar una función al evento `onclick` del botón `addToCartBtn` (dentro del modal):
                    *   Obtener la cantidad ingresada (`quantityInput.value`, convertir a número).
                    *   Validar que la cantidad sea mayor que 0.
                    *   Si es válida, llamar a `addItemToCart` con el vehículo y la cantidad.
                    *   Ocultar el modal.
            *   `addItemToCart` function:
                *   Buscar si el vehículo ya existe en el carrito (usando `codigo`).
                *   Si existe, incrementar la cantidad.
                *   Si no existe, añadir un nuevo objeto al array `cart`.
                *   Llamar a `updateCartUI`.
            *   `updateCartUI` function:
                *   Limpiar el contenido de `cartItemsContainer`.
                *   Iterar sobre el array `cart`.
                *   Para cada ítem, calcular el subtotal (precio * cantidad).
                *   Crear un elemento HTML (`<p>`) para mostrar la información del ítem (logo, imagen del auto, marca, modelo, cantidad, subtotal).
                *   Añadir el elemento a `cartItemsContainer`.
                *   Calcular el total del carrito.
                *   Actualizar el contenido de `cartTotalSpan` con el total formateado.
                *   Actualizar el contenido de `cartCount` con el número total de ítems en el carrito.

        *   **Simulación de Pago y Generación de Factura:**
            *   Añadir un event listener de tipo `click` al botón `processPaymentBtn`.
            *   Dentro del listener:
                *   Mostrar un mensaje de alerta indicando que el pago fue exitoso.
                *   Llamar a `generateInvoice`.
                *   Vaciar el array `cart`.
                *   Llamar a `updateCartUI`.
                *   Ocultar los modales de pago y carrito (usar `bootstrap.Modal.getInstance`).
            *   `generateInvoice` function:
                *   Crear una instancia de `jsPDF` (asegúrate de incluir la librería desde CDN en el HTML).
                *   Añadir texto al PDF: el nombre, título de la factura, fecha, detalles de los ítems del carrito (marca, modelo, cantidad, subtotal), total.
                *   Guardar el PDF con un nombre de archivo apropiado.

        *   **Event Listeners:**
            *   Añadir un event listener al evento `input` del `searchInput` para llamar a `filterVehicles`.
            *  Asegurar que todo el script se ejecute dentro de un listener para el evento `DOMContentLoaded`.
            * Llamar a la función `loadVehicles` para cargar los datos al inicio.

4.  **Especificaciones Técnicas Clave:**

    *   Uso de Bootstrap 5 para el layout y estilos base.
